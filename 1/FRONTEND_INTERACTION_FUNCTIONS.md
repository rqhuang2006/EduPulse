# 知行镜前端交互功能说明

本文档用于梳理当前前端交互程序中已经接入的页面、控件、跳转关系与功能能力，方便演示、答辩和后续继续扩展。

## 1. 入口与全局交互

### 1.1 左侧主侧边栏

左侧深棕色主侧边栏当前包含以下功能：

- `品牌区`
  - 显示系统名称 `知行镜`
  - 显示英文副标题 `A14 Student Insight`
  - 显示系统定位 `学生行为分析与干预工作台`

- `工作区`
  - 教师/管理端页面导航
  - 学生端页面导航
  - 当前页面高亮显示

- `全局筛选`
  - 风险等级筛选
  - 主导维度筛选
  - 行为模式筛选
  - 综合风险下限滑块

- `学生焦点`
  - 当前样例学生选择
  - 会影响学生档案、链路追踪、部分 AI 问答的默认对象

### 1.2 页面内跳转

程序内当前支持两类跳转：

- `侧边栏直接切页`
- `页面内部按钮跳转`

页面内部按钮主要出现在以下区域：

- 指标验证台中的“任务目标对齐”
- 指标验证台中的“技术指标验收”
- 成果总览中的“成果分组”
- 成果总览中的“重点入口”
- 成果总览中的“样例学生”

### 1.3 数据驱动逻辑

前端主要使用以下数据结果驱动展示：

- 融合主表 `master_df`
- 群体模式表 `pattern_df`
- 评估结果表 `eval_df`
- 学生级报告记录 `report_records`
- 前端打包产物 `frontend_bundle`
- 示例链路 `demo_case`

## 2. 教师/管理端页面

### 2.1 总览工作台

页面定位：学校端主工作台。

当前功能：

- 顶部学校端标题展示
- 核心指标卡片
  - 学生总数
  - 高风险人数
  - 平均综合风险
  - 主导风险维度
- `重点学生清单`
  - 展示高风险或优先级更高的学生
- `行为模式分布`
  - 展示群体行为模式数量或分布结果
- `三维风险区间`
  - 展示学习、生活、运动三个维度的整体风险情况
- `智能对话助手`
  - 选择学生
  - 输入问题
  - 点击提问
  - 快速追问按钮

### 2.2 成果总览

页面定位：将“8~10 个学生行为数据分析成果”按“成果类型”而不是按学生人数来组织展示。

当前功能：

- 顶部概览指标
  - 成果类型数量
  - 群体模式数量
  - 个体报告数量
  - 最佳 AUC
- `成果分组`
  - 群体洞察
  - 个体诊断
  - 预测干预
- 每项成果都支持：
  - 查看成果
  - 对应页面提示
  - 部分成果支持跳到样例学生
- `重点入口`
  - 一键进入总览工作台
  - 一键进入群体画像
  - 一键进入学生档案
  - 一键进入预警干预
  - 一键进入新增预测
  - 一键进入 AI 助手
- `样例学生`
  - 直接进入学生档案
  - 直接进入链路追踪
- `成果清单明细`
  - 列出全部成果类型、说明、展示页面与当前状态

当前已经纳入成果总览的 10 类成果包括：

1. 学校端总览工作台
2. 重点学生清单
3. 行为模式分布
4. 三维风险区间分析
5. 群体画像模式表
6. 学生个体数字档案
7. 精准预警干预清单
8. 新增学生风险预测
9. 解释链路与关键行为依据
10. 个性化评价与问答报告

### 2.3 指标验证台

页面定位：对照竞赛任务书中的用户期望、开发说明和技术指标进行展示。

当前功能：

- 顶部验收指标卡
  - 用户期望覆盖
  - 成果输出项
  - 行为模式发现
  - 最佳 AUC
- `任务目标对齐`
  - 逐条显示任务目标
  - 对应页面按钮可点击跳转
  - 部分目标支持直达样例学生
- `技术指标验收`
  - 显示每条验收项
  - 显示当前结果
  - 显示判定状态
  - 支持跳到相关页面
- `多源数据范围`
  - 按基础信息、学习过程、线上学习、生活轨迹、图书馆、运动体测、发展结果分组展示
- `分析成果清单`
  - 展示当前前端显式对应的分析成果类型
- `Harness 三域验收`
  - 展示学习、生活、运动三域相关验收结果
- `Harness 闭环概览`
  - 学生数量
  - 干预建议数
  - 群体模式数
  - 最近更新时间
- `展开工程明细`
  - 三域数据同步
  - 多 Agent 输出覆盖
  - 主线验收明细

### 2.4 群体画像

页面定位：展示学生群体层面的结构、模式与风险区间。

当前功能：

- 顶部群体画像标题
- 群体级指标卡
- `群体模式图`
  - 展示群体模式结构与分层
- `三维风险区间`
  - 展示学习、生活、运动三域分布
- `模式表`
  - 列出群体模式说明
- `群体说明`
  - 对当前群体特征进行文字说明

### 2.5 学生数字档案

页面定位：展示单个学生的画像、风险、解释与干预建议。

当前功能：

- 顶部学生档案标题
- 学生基础指标卡
  - 综合风险
  - 主导维度
  - MAP 机制
  - 干预类型
- `系统生成档案`
  - 展示学生级综合画像文字
- `三类解释视角`
  - 行为层
  - 心理/机制层
  - 环境影响层
- `三维风险`
  - 学习风险
  - 生活风险
  - 运动风险
- `档案标签与建议`
  - 风险等级标签
  - 模式标签
  - 干预建议
- `多Agent依据`
  - 读取行为解释
  - 读取机制解释
  - 读取干预解释

### 2.6 精准预警干预

页面定位：对高风险、重点关注学生进行筛查与干预展示。

当前功能：

- 顶部预警页面标题
- 预警人数和相关统计指标
- 预警学生清单
- 干预建议展示
- 结合当前筛选条件输出重点名单

### 2.7 新增预测

页面定位：支持输入新学生样本并进行风险预测。

当前功能：

- 顶部新增预测标题
- 输入预测所需字段
- 生成标准化入参
- 展示预测结果
- 展示模型或输出说明

### 2.8 AI 辅导员助手

页面定位：面向教师或辅导员的对话式智能助手。

当前功能：

- 历史对话展示
- 学生选择框
- 自然语言提问输入框
- 快捷问题按钮
- 根据当前筛选范围和样例学生生成回答
- 可回答的问题方向包括：
  - 当前最值得关注的是哪类学生
  - 当前高风险学生有什么特点
  - 为什么会被判定为高风险
  - 应该给出什么干预建议

### 2.9 链路追踪

页面定位：展示单个案例从三域风险到融合判断，再到解释与干预的完整链路。

当前功能：

- `步骤卡片`
  - 三域风险
  - 融合判断
  - 解释与干预
- `风险与融合`
  - 原始风险表
  - 融合结果表
- `解释与机制`
  - 解释线索
  - MAP 机制判断
  - 模式标签
- `链路摘要`
  - 输出综合说明
- `干预输出`
  - 输出建议文本

## 4. 当前页面跳转关系

### 4.1 指标验证台中的跳转

- 任务目标对齐 -> 可跳到对应模块页面
- 技术指标验收 -> 可跳到对应成果页面
- 样例学生按钮 -> 跳到学生数字档案

### 4.2 成果总览中的跳转

- 查看成果 -> 跳到该成果对应页面
- 样例学生 -> 跳到学生档案或链路追踪
- 重点入口 -> 快速进入主要工作页面

---

## 5. 当前解释体系

围绕“对关键问题提供不少于 3 个维度的解释”，前端当前采用以下解释结构：

- `行为证据`
  - 体现学生在学习、生活、运动、任务参与等方面的可观察表现

- `心理/机制归因`
  - 结合 MAP 机制
  - 关注动机、能力、提示支持中的主要短板

- `环境/支持影响`
  - 强调外部支持、任务结构、反馈频率、同伴与组织支持等因素

该三层结构已经在 `学生数字档案` 页面中显式展示。

## 6. 当前前端可支撑的答辩口径

### 6.1 关于“8~10 个学生行为数据分析成果”

当前更适合解释为：

- `8~10 类成果类型`
- 而不是 `8~10 个具体学生`

系统当前已经在“成果总览”中按成果类型组织出 10 类成果。

### 6.2 关于“3 个维度的解释”

当前更适合解释为：

- 行为层解释
- 心理/机制层解释
- 环境影响层解释

而不是简单理解为“学习、生活、运动”三个业务域。
